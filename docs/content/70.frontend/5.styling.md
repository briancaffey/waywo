# Styling

The waywo frontend uses Tailwind CSS 4.1+ as its styling foundation, with a custom theme configured through CSS custom properties to support both light and dark modes.

## Tailwind CSS Setup

Tailwind is loaded via the Vite plugin (not PostCSS) and configured in `frontend/app/assets/css/tailwind.css`:

```ts
// nuxt.config.ts
import tailwindcss from '@tailwindcss/vite'

export default defineNuxtConfig({
  css: ['~/assets/css/tailwind.css'],
  vite: {
    plugins: [tailwindcss()],
  },
})
```

The CSS file imports Tailwind and the animation library:

```css
@import "tailwindcss";
@import "tw-animate-css";
```

## Theme System

The theme uses CSS custom properties mapped to Tailwind color utilities via `@theme inline`. This is the shadcn/ui approach -- all colors are defined as CSS variables and referenced by Tailwind.

### Color Tokens

| Token | Purpose |
|-------|---------|
| `--background` / `--foreground` | Page background and default text |
| `--card` / `--card-foreground` | Card component colors |
| `--popover` / `--popover-foreground` | Popover and dropdown colors |
| `--primary` / `--primary-foreground` | Primary action color (buttons, links) |
| `--secondary` / `--secondary-foreground` | Secondary elements (badges, tags) |
| `--muted` / `--muted-foreground` | Muted backgrounds and subdued text |
| `--accent` / `--accent-foreground` | Accent/hover states |
| `--destructive` / `--destructive-foreground` | Destructive actions (delete, reset) |
| `--border` | Border color |
| `--input` | Input field borders |
| `--ring` | Focus ring color |
| `--chart-1` through `--chart-5` | Chart visualization colors |
| `--sidebar-*` | Sidebar-specific color variants |

### Color Format

Colors use the OKLCH color space for perceptual uniformity:

```css
:root {
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
}
```

### Border Radius

Radius values are derived from a single `--radius` base variable:

```css
:root {
  --radius: 0.625rem;
}
```

Tailwind utilities map to computed values:
- `rounded-sm` = `calc(var(--radius) - 4px)`
- `rounded-md` = `calc(var(--radius) - 2px)`
- `rounded-lg` = `var(--radius)`
- `rounded-xl` = `calc(var(--radius) + 4px)`

## Dark Mode

Dark mode is implemented via the `@nuxtjs/color-mode` module with a `dark` class on the `<html>` element.

### CSS Custom Variant

Tailwind 4 uses a custom variant to target dark mode:

```css
@custom-variant dark (&:is(.dark *));
```

### Dark Theme Colors

The `.dark` class overrides all color tokens:

```css
.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  --primary: oklch(0.985 0 0);
  --primary-foreground: oklch(0.205 0 0);
  /* ... all tokens inverted */
}
```

### Usage in Templates

Use Tailwind dark variant classes for per-element overrides:

```html
<div class="bg-green-50 dark:bg-green-950 border-green-200 dark:border-green-800">
  <span class="text-green-800 dark:text-green-200">Success</span>
</div>
```

## Tailwind Merge

The `tailwind-merge` package is used for resolving class conflicts when composing components. This is handled internally by shadcn/ui's `cn()` utility, which combines `clsx` and `tailwind-merge`:

```ts
import { clsx } from 'clsx'
import { twMerge } from 'tailwind-merge'

function cn(...inputs) {
  return twMerge(clsx(inputs))
}
```

## Base Styles

Global base styles are applied in the `@layer base` block:

```css
@layer base {
  * {
    @apply border-border outline-ring/50;
  }
  body {
    @apply bg-background text-foreground;
  }
}
```

This ensures all elements use the theme's border color and the body uses the theme background/foreground.

## Prose Styles

Custom prose styles in `tailwind.css` handle markdown content rendered from HN comments and chatbot responses:

```css
.prose p { @apply mb-3 last:mb-0; }
.prose h1 { @apply text-xl; }
.prose h2 { @apply text-lg; }
.prose ul { @apply list-disc mb-3 pl-5; }
.prose ol { @apply list-decimal mb-3 pl-5; }
.prose code { @apply bg-muted px-1.5 py-0.5 rounded text-sm font-mono; }
.prose pre { @apply bg-muted p-3 rounded-md overflow-x-auto mb-3; }
.prose blockquote { @apply border-l-4 border-muted-foreground/30 pl-4 italic mb-3; }
.prose a { @apply text-primary underline hover:no-underline; }
```

These styles are used with the `prose` class on containers rendering HTML via `v-html`:

```vue
<div class="prose prose-sm max-w-none dark:prose-invert" v-html="content" />
```

## Animations

The `tw-animate-css` package provides animation utilities. Custom animations for accordion open/close are defined in the theme:

```css
--animate-accordion-down: accordion-down 0.2s ease-out;
--animate-accordion-up: accordion-up 0.2s ease-out;
```

Spinner animations use Tailwind's built-in `animate-spin` class, applied to loading indicators throughout the application:

```html
<Icon name="lucide:loader-2" class="h-8 w-8 animate-spin text-muted-foreground" />
```

## Responsive Design

Pages use Tailwind responsive breakpoints for layout adaptation:

```html
<!-- Single column on mobile, multi-column on larger screens -->
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8">

<!-- Stack vertically on mobile, row on larger screens -->
<div class="flex flex-col sm:flex-row gap-4">
```

All page content is wrapped in a container with responsive padding:

```html
<div class="container mx-auto px-4 py-12">
  <div class="max-w-5xl mx-auto">
```

# Frontend Overview

The waywo frontend is a single-page application built with Nuxt 4 and Vue 3. It provides a complete interface for browsing Hacker News "What are you working on?" threads, exploring AI-extracted projects, performing semantic search, and chatting with a RAG-powered assistant.

## Application Entry Point

The root layout is defined in `app/app.vue`:

```vue
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>
```

All pages are rendered inside `<NuxtPage />` with `<NuxtLayout>` providing any shared layout structure.

## API Communication

Every page communicates with the Python backend via `$fetch` from Nuxt. The API base URL is configured through Nuxt runtime config:

```ts
const config = useRuntimeConfig()
// Usage: $fetch(`${config.public.apiBase}/api/endpoint`)
```

The default API base URL is `http://localhost:8008`. Override it by setting the `NUXT_PUBLIC_API_BASE` environment variable.

## Development

```bash
cd frontend

# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build

# Preview production build
npm run preview
```

## Key Architectural Decisions

- **No SSR data fetching** -- All API calls happen client-side in `onMounted` hooks using `$fetch`.
- **File-based routing** -- Pages live in `app/pages/` and map directly to URL routes.
- **shadcn component pattern** -- Base UI components in `components/ui/` are generated via the shadcn CLI and composed into page-level layouts.
- **Custom component prefix** -- Components in `components/waywo/` are auto-imported with a `Waywo` prefix (e.g., `PostsChart.vue` becomes `<WaywoPostsChart />`).
- **TypeScript throughout** -- All `<script setup>` blocks use `lang="ts"` with inline interface definitions.

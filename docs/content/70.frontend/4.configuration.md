# Configuration

All frontend configuration is centralized in `frontend/nuxt.config.ts`.

## Full Configuration

```ts
import tailwindcss from '@tailwindcss/vite'

export default defineNuxtConfig({
  compatibilityDate: '2025-07-15',
  components: [
    {
      path: '~/components/waywo',
      prefix: 'Waywo',
      extensions: ['.vue'],
    },
  ],
  css: ['~/assets/css/tailwind.css'],
  vite: {
    plugins: [tailwindcss()],
  },
  devtools: { enabled: true },
  runtimeConfig: {
    public: {
      apiBase: 'http://localhost:8008',
    }
  },
  modules: [
    '@nuxt/image',
    '@nuxt/icon',
    '@nuxt/fonts',
    '@nuxt/eslint',
    ['@nuxtjs/color-mode', { classSuffix: '' }],
    'shadcn-nuxt'
  ],
  shadcn: {
    prefix: '',
    componentDir: './app/components/ui'
  }
})
```

## Runtime Config

### `runtimeConfig.public.apiBase`

The base URL for all backend API calls. Defaults to `http://localhost:8008`.

**Override at runtime** with the `NUXT_PUBLIC_API_BASE` environment variable:

```bash
# Development
NUXT_PUBLIC_API_BASE=http://localhost:8008 npm run dev

# Production
NUXT_PUBLIC_API_BASE=https://api.example.com npm run build
```

**Usage in components:**

```ts
const config = useRuntimeConfig()
const data = await $fetch(`${config.public.apiBase}/api/waywo-posts`)
```

## Modules

| Module | Purpose |
|--------|---------|
| `@nuxt/image` | Optimized image handling and transformations |
| `@nuxt/icon` | Icon component with Lucide and other icon sets |
| `@nuxt/fonts` | Automatic font optimization and loading |
| `@nuxt/eslint` | ESLint integration for code quality |
| `@nuxtjs/color-mode` | Light/dark theme switching with `classSuffix: ''` |
| `shadcn-nuxt` | Auto-imports shadcn/ui components from `components/ui/` |

### Color Mode Configuration

The color mode module is configured with an empty `classSuffix`:

```ts
['@nuxtjs/color-mode', { classSuffix: '' }]
```

This means the dark mode class applied to `<html>` is just `dark` (not `dark-mode`), which aligns with Tailwind CSS conventions and the shadcn theming system.

### shadcn Module Configuration

```ts
shadcn: {
  prefix: '',                          // Components use bare names: <Button>, <Card>
  componentDir: './app/components/ui'  // Output directory for generated components
}
```

## Component Auto-Import

### Default (ui components)

All components in `components/ui/` are auto-imported by the `shadcn-nuxt` module. No explicit imports needed.

### Custom (waywo components)

Components in `components/waywo/` are auto-imported with the `Waywo` prefix:

```ts
components: [
  {
    path: '~/components/waywo',
    prefix: 'Waywo',
    extensions: ['.vue'],
  },
]
```

For example, `components/waywo/PostsChart.vue` becomes `<WaywoPostsChart />` in templates. However, components can also be explicitly imported when needing a ref:

```ts
import PostsChart from '~/components/waywo/PostsChart.vue'
```

## Vite Configuration

Tailwind CSS is integrated via the Vite plugin rather than PostCSS:

```ts
import tailwindcss from '@tailwindcss/vite'

vite: {
  plugins: [tailwindcss()],
}
```

## Development Tools

Nuxt DevTools are enabled for development:

```ts
devtools: { enabled: true }
```

This provides an in-browser panel for inspecting components, routes, modules, and runtime state.

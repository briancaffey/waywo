# Pages

The waywo frontend has 12 pages organized using Nuxt file-based routing. All pages live in `frontend/app/pages/`.

## Route Map

| Route | File | Description |
|-------|------|-------------|
| `/` | `index.vue` | Landing page with hero section, feature cards, and CTAs |
| `/posts` | `posts.vue` | View HN posts, trigger processing, bar chart visualization |
| `/comments` | `comments/index.vue` | Paginated comment browser with post filter |
| `/comments/[id]` | `comments/[id].vue` | Single comment detail with extracted projects |
| `/projects` | `projects/index.vue` | Browse and filter projects with 10+ filter options |
| `/projects/[id]` | `projects/[id].vue` | Full project detail with scores, URLs, tags, logs |
| `/search` | `search.vue` | Semantic search with similarity scores |
| `/chat` | `chat.vue` | RAG chatbot for project Q&A |
| `/admin` | `admin.vue` | Database stats, service health, reset tools |
| `/prompts` | `prompts.vue` | Inspect workflow step prompt templates |
| `/voice` | `voice.vue` | Voice-to-voice chat with the AI assistant |
| `/debug` | `debug.vue` | Test backend API with a debug Celery task |

## Page Details

### Home (`/`)

The landing page with a hero section, six feature cards (Semantic Search, AI Chat, AI Extraction, Filter & Explore, Monthly Threads, Voice Search), and a call-to-action section linking to `/chat` and `/posts`.

### Posts (`/posts`)

Displays monthly "What are you working on?" posts in a table with date, title, score, and comment counts. Includes controls to trigger post processing from the HN API with configurable limits. Features a `PostsChart` bar chart (Unovis) showing comments per post with a custom tooltip.

**Key interactions:**
- Trigger post fetching from HN API with optional post/comment limits
- Click a post row to navigate to its comments
- Click comment count badge to filter comments by post

### Comments (`/comments`)

Paginated list of HN comments with author avatars, timestamps, and HTML content rendered inline. Supports filtering by `post_id` query parameter.

**Key interactions:**
- "Extract Projects" button to batch-process all comments
- Per-comment "Process" button for individual extraction
- Navigate to comment detail or view extracted projects
- Pagination with 20 items per page

### Comment Detail (`/comments/[id]`)

Shows full comment content, parent post info, extracted projects list, and metadata (ID, parent, replies, type). Each extracted project card shows title, validity badge, scores, and hashtags.

### Projects (`/projects`)

The most feature-rich page. Displays project cards with title, description, scores, hashtags, and timestamps. Projects can be bookmarked and deleted.

**Filters (collapsible panel):**
- Tag multi-select with autocomplete (fetched from API)
- Idea score range slider (1-10)
- Complexity score range slider (1-10)
- Date from / date to inputs
- Bookmarked toggle
- Comment ID filter (via query param)

Filter state is synced to URL query parameters so filtered views can be shared or bookmarked.

### Project Detail (`/projects/[id]`)

Full project view with:
- Bookmark toggle and validity badge
- Idea score and complexity score display
- Hashtag badges
- Full description card
- Collapsible URLs section with scraped content summaries
- Collapsible original HN comment with author info
- Collapsible processing workflow logs
- Metadata card (IDs, timestamps)
- Delete button with confirmation

### Search (`/search`)

Semantic search powered by vector embeddings. Shows embedding service health status and coverage stats. Results display similarity percentage with color-coded badges (Excellent/Great/Good/Fair/Partial).

**API:** `POST /api/semantic-search` with `{ query, limit }` body.

### Chat (`/chat`)

RAG chatbot interface with:
- Scrollable message history (500px container)
- User and assistant message bubbles
- Markdown rendering via `marked` (GFM enabled, line breaks enabled)
- Source project links with similarity percentages
- Clickable suggestion prompts for new users
- Clear chat button

**API:** `POST /api/waywo-chatbot` with `{ query, top_k }` body.

### Admin (`/admin`)

System administration dashboard with three sections:

1. **External Services** -- Health checks for LLM server, embedding server, and rerank server
2. **Database Statistics** -- Counts for posts, comments, projects, embeddings, and Redis status
3. **Danger Zone** -- Reset SQLite, reset Redis, or reset everything (with double confirmation)

Also includes links to rebuild the vector index and view workflow prompts.

### Prompts (`/prompts`)

Lists all workflow step prompt templates from the backend. Each step card shows step number, title, description, workflow name, input/output event types, template variables, and the full prompt template text.

### Voice (`/voice`)

Voice-to-voice chat with the AI assistant. Uses WebSocket for real-time bidirectional audio and a hold-to-talk interaction model. Conversations are persisted as threads with full history.

**Key features:**
- Hold-to-talk button with animated state indicators (listening, processing, speaking)
- Real-time partial transcription while speaking
- Conversation history in scrollable message bubbles (user right, assistant left)
- Thread sidebar (left) for managing multiple conversations
- Debug panel (right) with real-time event telemetry and timing metrics
- TTS voice selection dropdown (grouped by language)
- Text display toggle

**WebSocket:** `/ws/voice-chat?thread_id=<uuid>`
**REST APIs:** `GET/POST /api/voice/threads`, `GET/PUT/DELETE /api/voice/threads/{id}`, `GET /api/voice/voices`

::tip
See [Voice Mode](/voice-mode/overview) for full architecture, protocol, and API documentation.
::

### Debug (`/debug`)

Minimal page with a single button to trigger a debug Celery task (sleeps for 1 second). Used to verify backend API and task queue connectivity.

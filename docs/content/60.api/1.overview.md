# API Overview

The Waywo backend exposes a REST API built with FastAPI. All endpoints return JSON and follow consistent conventions for pagination, error handling, and response structure.

## Base URL

```
http://localhost:8008
```

The backend runs on port **8008** by default inside Docker Compose. All API routes are prefixed with `/api/`.

## Interactive Documentation

FastAPI auto-generates two interactive documentation UIs from the OpenAPI schema:

| UI | URL | Description |
|---|---|---|
| Swagger UI | [http://localhost:8008/docs](http://localhost:8008/docs) | Interactive "try it out" interface |
| ReDoc | [http://localhost:8008/redoc](http://localhost:8008/redoc) | Read-only reference documentation |
| OpenAPI JSON | [http://localhost:8008/openapi.json](http://localhost:8008/openapi.json) | Raw OpenAPI 3.x specification |

## Response Format

All responses are JSON. Successful responses return the data directly. Error responses follow the standard FastAPI format:

```json
{
  "detail": "Post 999 not found"
}
```

Common HTTP status codes:

| Code | Meaning |
|---|---|
| `200` | Success |
| `400` | Bad request (missing or invalid parameters) |
| `404` | Resource not found |
| `500` | Internal server error |
| `503` | Dependent service unavailable (LLM, embedding, rerank) |

## Pagination

List endpoints use an **offset/limit** pattern:

| Parameter | Type | Default | Description |
|---|---|---|---|
| `offset` | `int` | `0` | Number of records to skip |
| `limit` | `int` | `50` | Maximum number of records to return |

Paginated responses include metadata alongside the results:

```json
{
  "comments": [...],
  "total": 1200,
  "limit": 50,
  "offset": 0
}
```

Use `total` to calculate the number of pages: `ceil(total / limit)`.

## Authentication

No authentication is required. The API is designed for local or trusted-network deployment and does not enforce any auth mechanism.

## CORS

All origins are allowed. The backend is configured with a permissive CORS policy:

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

This allows the Nuxt frontend (or any other client) to call the API from any domain.

## Health Check

A simple health check endpoint is available at the root API level:

```
GET /api/health
```

**Response:**

```json
{
  "status": "ok"
}
```

# Generate Ideas

The generation API triggers and monitors synthetic project idea generation powered by NeMo Data Designer. It runs as an async Celery task with polling-based progress tracking.

::tip
See [Data Designer](/data-designer/overview) for the full pipeline architecture.
::

## Create Generation Task

Enqueue a new idea generation task.

```
POST /api/generate-ideas
```

**Request Body:**

| Field | Type | Default | Description |
|---|---|---|---|
| `num_ideas` | `int` | `5` | Number of ideas to generate (1--50) |
| `seed_tags` | `string[] \| null` | `null` | Tags to guide generation; omit for full vocabulary |
| `creativity` | `float` | `0.85` | LLM temperature (0.1--1.5) |

**Example Request:**

```bash
curl -X POST http://localhost:8008/api/generate-ideas \
  -H "Content-Type: application/json" \
  -d '{"num_ideas": 5, "seed_tags": ["nutrition", "ios"], "creativity": 0.85}'
```

**Response:** `200 OK`

```json
{
  "task_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "num_requested": 5,
  "seed_tags": ["nutrition", "ios"]
}
```

## Poll Task Status

Check the progress or result of a generation task.

```
GET /api/generate-ideas/{task_id}/status
```

**Path Parameters:**

| Parameter | Type | Description |
|---|---|---|
| `task_id` | `string` | Celery task ID from the create endpoint |

**Response varies by state:**

::code-group
```json [PENDING]
{
  "task_id": "a1b2c3d4-...",
  "state": "PENDING"
}
```

```json [STARTED]
{
  "task_id": "a1b2c3d4-...",
  "state": "STARTED",
  "stage": "generating",
  "progress": 3,
  "total": 5
}
```

```json [SUCCESS]
{
  "task_id": "a1b2c3d4-...",
  "state": "SUCCESS",
  "result": {
    "status": "success",
    "num_requested": 5,
    "num_generated": 5,
    "num_saved": 5,
    "project_ids": [101, 102, 103, 104, 105],
    "errors": [],
    "seed_tags": ["nutrition", "ios"]
  }
}
```

```json [FAILURE]
{
  "task_id": "a1b2c3d4-...",
  "state": "FAILURE",
  "error": "Error generating dataset: LLM endpoint unreachable"
}
```
::

**Stage values during STARTED:**

| Stage | Description |
|---|---|
| `building_pipeline` | Loading projects, computing tag stats, configuring NDD |
| `generating` | NDD is running LLM calls to produce records |
| `saving` | Post-processing rows: parsing, embedding, saving to DB |

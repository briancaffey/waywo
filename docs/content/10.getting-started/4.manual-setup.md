# Manual Setup

This guide walks through running WAYWO directly on your machine without Docker. This is useful for development where you want faster feedback loops, debugger access, and hot-reloading.

## Prerequisites

- Python >= 3.12
- Node.js >= 20
- Redis running locally (or accessible on your network)
- AI model endpoints accessible (see [Prerequisites](/getting-started/prerequisites))

## Backend Setup

::steps

### Install Python dependencies

From the project root:

```bash
pip install -e ".[dev,notebook]"
```

This installs WAYWO in editable mode along with dev tools (pytest, black, mypy) and Jupyter Lab.

### Set environment variables

Export the required environment variables for your shell session. Adjust the URLs to match your setup:

```bash
export CELERY_BROKER_URL=redis://localhost:6379/0
export CELERY_RESULT_BACKEND=redis://localhost:6379/0
export REDIS_URL=redis://localhost:6379/0
export EMBEDDING_URL=http://<your-embedding-host>:8000
export RERANK_URL=http://<your-rerank-host>:8111
export LLM_BASE_URL=http://<your-llm-host>:8002/v1
export LLM_MODEL_NAME=nvidia/NVIDIA-Nemotron-3-Nano-30B-A3B-BF16
export DATA_DIR=./data
export LOG_LEVEL=DEBUG
export PHOENIX_HOST=localhost
export PHOENIX_PORT=6006
export NAT_SERVICE_URL=http://localhost:8080
export FIRECRAWL_URL=http://localhost:3002
```

### Run database migrations

```bash
python -m src.migrate
```

### Start Redis

If Redis is not already running:

```bash
# Using Docker (simplest approach)
docker run -d --name waywo-redis -p 6379:6379 -p 8001:8001 redis/redis-stack:latest

# Or using your system Redis
redis-server
```

### Start the Celery worker

```bash
celery -A src.celery_app worker --loglevel=info --queues=waywo
```

::note
The Docker setup uses `--concurrency=1` and wraps the worker in `watchmedo auto-restart` for file-watching. For development, you can add `--concurrency=1` if needed, or use watchmedo similarly.
::

### Start Celery Beat (scheduler)

In a separate terminal:

```bash
celery -A src.celery_app beat --loglevel=info
```

### Start the backend server

In another terminal:

```bash
uvicorn src.main:app --host 0.0.0.0 --port 8000 --reload
```

The API will be available at [http://localhost:8000](http://localhost:8000) with Swagger docs at [http://localhost:8000/docs](http://localhost:8000/docs).

::tip
When running manually, the backend listens on port **8000** directly. In the Docker setup, port 8000 inside the container is mapped to **8008** on the host.
::

::

## Frontend Setup

::steps

### Install Node.js dependencies

```bash
cd frontend
npm install
```

### Start the development server

```bash
npm run dev
```

The Nuxt 4 frontend will start with hot module replacement enabled.

::

## Optional Services

### Flower (Celery monitoring)

```bash
celery -A src.celery_app flower --port=5555
```

### Jupyter Lab

```bash
jupyter lab --port=8888 --no-browser --notebook-dir=./notebooks
```

### Phoenix (LLM observability)

Phoenix runs as a standalone container even in manual setups:

```bash
docker run -d --name waywo-phoenix -p 6006:6006 -p 4317:4317 arizephoenix/phoenix:latest
```

## Code Formatting

The project uses black for code formatting. You can run it via the Makefile:

```bash
make black     # Format code
make check     # Check formatting without changes
```
